## Per-Feature Details

### CLI (build, run, output generation)

- **Role:** Computational physicist setting up mVMC simulations
- **What they tried:** Build, run example file, run README example, edge cases (missing file, invalid input, --help, no args), inspect output files
- **Discoverability:** README is clear for the Hubbard model use case. Could build and run on first try. However, Heisenberg model support (Sp, Sm, Sz operators) is not mentioned in the README at all -- only discoverable via CLAUDE.md or source code.
- **Setup:** `cargo build --release` succeeded. Three dead-code warnings (unused methods `site()`, `num_ops()`, unused field `dimension`).
- **Functionality:** All 46 tests pass. CLI works correctly for:
  - Example file (hubbard_2site.qsl) -- 5 terms, correct output
  - README 10-site Hubbard model with PBC -- 50 terms, 40 transfers, 10 interactions
  - Error cases: missing file (clear error, exit 1), invalid input (parse error, exit 1), no args (clap usage, exit 2)
  - Output files (trans.def, interall.def, modpara.def, jastrowidx.def, gutzwilleridx.def) all verified correct
- **Friction points:**
  1. `namelist.def` has inconsistent whitespace -- first line has extra leading spaces
  2. No `--version` flag (trivial fix: add `#[command(version)]`)
  3. No `--verbose`/`--quiet` flags for controlling diagnostic output
  4. No validation when sum range exceeds lattice size with `pbc=false` -- silently drops terms
  5. File extension convention unclear (`.qsl` in example vs `.def` in README)
- **Doc suggestions:**
  1. Add Heisenberg model example to README
  2. Document all supported operators (c, n, Sp, Sm, Sz) explicitly
  3. Clarify file extension convention
  4. Document supported lattice dimensions (only 1d?)
  5. Add note about PBC vs OBC and sum range requirements

## Issues Found

1. **namelist.def formatting inconsistency** — first line has extra leading whitespace vs subsequent lines (in `src/output/mvmc.rs`)
2. **3 dead-code warnings** — unused methods `site()`, `num_ops()` and unused field `dimension`
3. **No --version flag** — one-line fix in `src/main.rs`
4. **Silent behavior with out-of-range indices** — with `pbc=false`, terms referencing nonexistent sites are silently dropped with no warning

## Suggestions

1. Fix namelist.def whitespace formatting (bug)
2. Add `#[command(version)]` to Cli struct (trivial)
3. Add Heisenberg example and operator reference to README (documentation gap)
4. Emit a warning when terms are dropped due to out-of-range indices in OBC mode (usability)
5. Clean up dead-code warnings (code quality)
