## Per-Feature Details

### Standard Hubbard Pipeline
- **Role:** Computational physics grad student
- **What they tried:** Build project, create 10-site Hubbard input, generate mVMC files
- **Discoverability:** Good. README Quick Start is clear and sufficient.
- **Setup:** cargo build --release works without issues
- **Functionality:** All 9 output files generated correctly. 40 hopping terms, 10 U terms. Physically correct.
- **Friction points:** Output file column formats not documented; modpara.def defaults unexplained; no "next step: run mVMC" guidance
- **Doc suggestions:** Document output column formats; add mVMC execution step

### Heisenberg Model
- **Role:** Condensed matter physicist studying spin chains
- **Discoverability:** Good for Sp/Sm/Sz operators. S(i).S(j) syntax only documented in correlation section.
- **Setup:** Works
- **Functionality:** Spin operators work correctly. S(i).S(j) works in both pipelines.
- **Issues found:**
  1. **CRITICAL: interall.def coefficient precision bug** — `{:.1}` format truncated 0.25 to 0.2 (20% error). **FIXED:** Changed to `{:.15}`.
  2. **README Heisenberg example was physically incorrect** — Missing 0.5 factor on Sp*Sm terms for isotropic model. **FIXED:** Changed to use `S(i).S(i+1)` shorthand.
- **Doc suggestions:** Add S(i).S(j) to operator table; note difference between explicit Sp/Sm and S.S shorthand

### YS Transform
- **Role:** Researcher applying particle-hole transformation
- **Discoverability:** Well-documented with dedicated README section
- **Setup:** Works
- **Functionality:** All correct — coulombintra.def created, namelist.def references it, term classification works
- **Friction points:** "offset=-0" display; LaTeX notation doesn't render on GitHub; no explanation of practical benefits
- **Doc suggestions:** Use ASCII notation instead of LaTeX; add before/after comparison example

### Correlation Pipeline
- **Role:** Physicist measuring spin-spin correlations
- **Discoverability:** Good. Example and output file table are clear.
- **Setup:** Works
- **Functionality:** S(i).S(j) and n(i,s) correlations both work. Both-pipelines mode works.
- **Friction points:** cisajscktaltdc.def contains no coefficients — user must manually combine with correlation_summary.txt for post-processing. This workflow is undocumented.
- **Doc suggestions:** Explain post-processing workflow; add S(i).S(j) to operator table
